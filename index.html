حساب صحة مفتاح الحساب الجاري البريدي (CCP Key)

يقوم النظام بالتحقق الحسابي من صحة رقم الحساب الجاري للموظف ومطابقته للمفتاح (Cle CCP) لتجنب تحويل أجور الموظفين والأساتذة شهرياً لحسابات خاطئة عن طريق الخطأ.

المعادلة والخوارزمية المطبقة (Modulo 97):

$calculated_key = 97 - (intval($ccp) * 100) % 97; return intval($key) === $calculated_key;

الجداول المتأثرة: Fonctionnaire, Encadrement

البيداغوجيا

مطابقة شروط السن القانوني للمتربصين (Age constraints)

يفرض النظام قانون التمهين الجزائري وقوانين التعليم البيداغوجي للقبول الفوري في عروض التكوين المتوفرة:

  • نمط التمهين (Apprentissage): يجب أن يتراوح عمر المتمهن بين 15 و 35 سنة عند تاريخ التعاقد.
  • النمط الحضوري (Résidentiel): يجب ألا يقل السن عن 16 سنة كاملة.
$age = Carbon::parse($date_naiss)->age; if ($mode === 'apprentissage' && ($age < 15 || $age > 35)) { throw new Exception("السن القانوني لعقد التمهين يجب أن يتراوح بين 15 و 35 سنة."); }

الجداول المتأثرة: Candidat, Preinscrit, Apprenant

الرقابة المالية

رقابة أرصدة بنود الميزانية السنوية (Budget article limits)

يمنع النظام التزام المؤسسة بنفقة مالية أو فاتورة مورد تفوق الاعتماد المالي المخصص والسنوي المحدد للمادة المحاسبية من الوزارة لمنع الوقوع في عجز الميزانية.

المعادلة البرمجية للتحقق:

if (($requested_amount + $current_commited) > $allocation) { throw new Exception("الاعتمادات المالية غير كافية للالتزام بهذه النفقة."); }

الجداول المتأثرة: BudgetArticle, Commitment

العقود والقوانين

كوتا عقود التمهين لدى الشركات المستخدمة (Employer Quota)

يتحقق النظام من عدم تخطي المؤسسات الاقتصادية للعدد القانوني للمتمهنين بناءً على حجم القوى العاملة المصرح بها في كشوف الضمان الاجتماعي (CNAS) للشركة لضمان جودة التأطير.

// التحقق من توافق نسبة المتمهنين مع العمال المسجلين if ($active_apprentices >= $employeur->max_quota_allowed) { throw new Exception("تجاوزت المؤسسة المستخدمة الكوتا المخصصة لها قانوناً."); }

الجداول المتأثرة: Employeur, Candidat_contratapp
