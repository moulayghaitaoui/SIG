حساب صحة مفتاح الحساب الجاري البريدي (CCP Key)

يقوم النظام بالتحقق الحسابي من صحة رقم الحساب الجاري للموظف ومطابقته للمفتاح (Cle CCP) لتجنب تحويل أجور الموظفين والأساتذة شهرياً لحسابات خاطئة عن طريق الخطأ.

المعادلة والخوارزمية المطبقة (Modulo 97):

$calculated_key = 97 - (intval($ccp) * 100) % 97; return intval($key) === $calculated_key;

مطابقة شروط السن القانوني للمتربصين (Age constraints)

يفرض النظام قانون التمهين الجزائري وقوانين التعليم البيداغوجي للقبول الفوري في عروض التكوين المتوفرة:

  • نمط التمهين (Apprentissage): يجب أن يتراوح عمر المتمهن بين 15 و 35 سنة عند تاريخ التعاقد.
  • النمط الحضوري (Résidentiel): يجب ألا يقل السن عن 16 سنة كاملة.
$age = Carbon::parse($date_naiss)->age; if ($mode === 'apprentissage' && ($age < 15 || $age > 35)) { throw new Exception("السن القانوني لعقد التمهين يجب أن يتراوح بين 15 و 35 سنة."); }

التكامل الرقمي الفوري مع منصة "مهنتي" الوطنية (Real-time Mihnati API)

الوضع الحالي: تتم المزامنة حالياً بطريقة غير مباشرة عبر استيراد ملفات مرجعية ثابتة بصيغة JSON (الولايات والبلديات والتخصصات).

الحل المقترح: تطوير بوابة ربط رقمية مباشرة (RESTful API / GraphQL) محمية بمفاتيح مرور مشفرة (API Keys / OAuth2) لتسهيل جلب ملفات المترشحين المقبولين نهائياً بصفة لحظية دون تدخل بشري وتحديث عروض التكوين المفتوحة بالمركز على المنصة الوطنية فور اعتمادها داخلياً.

نظام التنبيهات النصية والبريدية الفورية (Multi-channel Notification System)

الحل المقترح: دمج خدمات الإرسال الجماعي للرسائل النصية (Bulk SMS Gateways) الخاصة بالمتعاملين المحليين (Mobilis, Djezzy, Ooredoo) والبريد الإلكتروني لإعلام أولياء المتربصين فور تسجيل غياب غير مبرر لمنع التسريح التلقائي وإعلام المتمهنين والشركات بمواعيد تجديد أو توقيع عقود التمهين.

الحل المقترح: دمج محرك تقارير متطور في Laravel 12 مثل Browsershot (Puppeteer) أو Laravel Snappy (wkhtmltopdf) لضمان توليد ملفات PDF متطابقة 100% مع النماذج الرسمية لوزارة التكوين والتعليم المهنيين، مع طباعة الصكوك وحوالات الدفع البريدية CCP بترميز عربي سليم.
